Unit One.Zero

The Internet

People with computers download browsers; browsers, using HTTP, send out queries and return web pages plucked from super-space-saving filing-cabinet computers called servers...hidden in air conditioned closets.

These pages are usually formatted using HTML, hypertext markup language, and can contain almost anything: text, images, videos, links, pdf's, multiple levels. The browser reads the text, markup and references to turn the information into something user-friendly.

Elements

Elements are used in markup, which controls how a webp age, and its content, are rendered by the browser. Elements consist, usually, of an opening tag, content, and a closing tag. Some elements, called void tags, don't need closing tags.

Void Tags: Line break: <br> and the image element: <img src="url" alt="text"> Non-Void Tags: Bold, and/or Strong: <b>text</b>, <strong>text</strong>

Elements can be either inline or block. Inline elements create no additional space around them and can "butt" right up against text. Block elements create a box in which their content resides.

Line break <br> is inline, but paragraph <p></p>

is block text

and created the space preceding and following the above line.

But both are necessary to manage the "white space" on an HTML page, since spaces and carriage returns are rendered null when a browser reads HTML. Multiple spaces and any carriage returns are turned into a single space. Space, therefore, has to be deliberately coded in.

Attributes, span/div
Attributes modify their elements, or are properties of elements. Span and Div, for instance, can have attributes added to them. Both elements contain text, Span is inline and Div is block. Neither are void tags. Div behaves much like the paragraph tag, but doesn't create as much space around it.

Unit One.Two

Learning Code from Other Pages

Looking at the code of existing web pages provides excellent examples of code-in-action. Not only can you see the end product, but how the elements and style are formatted correctly to follow HTML (or CSS) syntax and rules.

Google Chrome's developer tools provides user-friendly ways to sift through complex HTML structure, as well as see how coders are minimizing redundancy (and errors). It's called a tool because not only can you use it, but through that use you add coding skill to your belt, so to speak.

HTML structure can be complex due to the "tree-like" nature of it. Within elements lie additional elements. This causes the wavy pattern we see when looking at HTML code; the further indented something is, the deeper it's nested. And, when looking at the page itself, this nesting is rendered as boxes within boxes (though the borders of the boxes may be invisible).

Programmer's text editors also make learning/performing programming easier. Programs for writing programs! Or web pages, in this case. They will, for instance, provide a corresponding closing tag for your opening one.

Boxes

Everything on a web page falls within the confines of a box. There's a standard model for the creation and formatting of boxes.

The padding, border, and margin all have the capability of being customized with differing sizes. The total size of the box is from top margin to bottom, and right to left. So, margin width (or height) plus border width plus padding width plus content width. These values can be set in pixels, or percentages. Percentages change with the screen size, but pixels are fixed. If using pixels, the max width option can help maintain the integrity of a box. Speaking of integrity, it's important to make sure boxes will be rendered correctly across browsers. Browser specific prefixes such as webkit, moz, and ms with the style definition will help.

Boxes must be positioned on the page as well. Without defining the box values, the box will go the whole width of a page. Using a flex box will allow boxes to sit next to each other. Manipulating individual values will also help place boxes.

It's also helpful to think of block elements as creating boxes around them, while inline elements go within boxes. Some box elements have to be structured within one another as well. The paragraph element, for instance, has to be within div(ision?) elements. If you try to put a div within a paragraph, your code will not verify though the browser may be able to render it how you intended.

Unit One.Three

CSS

As mentioned previously, CSS controls how an HTML page looks. It also saves coders substantial time by allowing them to apply a repetitive style to multiple elements of the same type.

There are three ways to add style to a page:
1. Writing it in the head element of the HTML page.
2. Within the head element, add a link to a separate CSS file. This keeps large projects organized, and looks something like this:

<link: rel="stylesheet" type="text/css" href="main.css">

3. Writing the style inline with the HTML. Not the best choice, leads to a lot of repetition.

Some Particulars of CSS

CSS both helps avoid repetition and maintain consistency by applying a singular style to an unlimited number of instances of the same element. This, handily enough, also makes implementing changes easier. It's also ruled by an abstract concept called inheritance. In a page of CSS code, the most specific rule applies. If a class has a specific attribute, and there's a class nested within it without any attributes specified, the nested class shares the attribute of the parent (ancestor) class.

A CSS Element: h1 { background-color: black; } "h1" is the selector. "background-color: black;" is the declaration "background-color is the property and "black" is the value

Grammar

Programming languages require strict adherence to their grammar rules or the computer doesn't know how to interpret them. Computers can't guess at the intended meaning. It will return a syntax error.

Backus-Naur Form: <Non-Terminal> --> replacement

Notation used to describe a programming language. The replacement can be any sequence of zero or more non-terminals and terminals. Terminals never appear on the left side. They are the end of the line, not replaceable with anything else.

Unit Two.Two

Variables

Variables in programming are similar to variables in algebra. A value is assigned a designation, the name composed of underscores, letters, and numbers (though it can't start with a number), and until that designation is assigned another value, the name can be used in expressions to represent that value. For the computer to understand the use of a variable, a programmer has to use an assignment statement.

Assignment Statement Name = Expression speed_of_light = 299792458 billionth = 1.0 / 1000000000 nanostick = speed_of_light * billionth * 100 And the command "print nanostick" will direct the computer to execute the equation: 299792458 * (1.0 / 1000000000) * 100

Some benefits of using variables include improving code readability, storing a singular value of data, and a way to change that value. The fewer times you have to write out 9 digit numbers, the fewer chances for error.

Not only can variables be used on either side of the equation, as evinced above, but the same variable can be used on either side.

Introduction to Strings

A string is a sequence of characters contained within quotes--single or double. It must start and end with the same type of quote character, but can contain the other type within it. Keep an eye on how the python interpreter changes the color when strings are inserted. Strings can be assigned to variables.

Concatenation: Using the operator (+) to link things together.

Strings can be added to one another, and they can be multiplied by integers, but they cannot be added to integers. However, they can be concatenated if the number is turned into a string: str(n) + "string"

.find

.find will find a character within a string. This returns the location where the character, or the first character in a sequence of them, is. So "E_of_E_String_E".find("E") will return 0. "E" occurs in the first character of the string. And "E_of_E_String_E".find("E", 1) will return the location of the second instance of the search term--5. Switching out the 1 for a 6 will return the location of the third instance. Storing a location within another string as a variable makes it reusable.

the_string = "The best dog ever: Penelope" dog_name_location = the_string.find("Penelope") best_dog = the_string[dog_name_location:] print best_dog + "! " + "The best dog is " + best_dog + "." >>> Penelope! The best dog is Penelope.

Unit Two.Three

Functions

A function takes input, works with it, and then produces output. Structurally, a function must be written with the starting term def, given parameters contained within parentheses, specifications as to what must be done with the parameters, and have a return command. Without a return command, the computer will not produce anything.

When calling a function, values must be specified to replace the parameters and it must be called by name. This way, a programmer can call a function repetitively without having to rewrite the entire process.

def cube(x): answer = x * x * x return answer print cube(5)s >>>125

If statements are evaluated for veracity, and if they're true, they run, but if not, they return false. They're one and done, so to speak. But loops, on the other hand, will continue to run until they're false, or encounter a break. A break will stop the loop even if the expression is true.

When calling a function, values must be specified to replace the parameters and it must be called by name. This way, a programmer can call a function repetitively without having to rewrite the entire process.

Control Flow & Loops: Using Or and-per-se And

Two other helpful keywords/options are Or & And. An Or code only evaluates as necessary; the code will return a true result as soon as it hits a true statement, even if the next possible option is an error. However, it will not return a false result if it encounters an error while continuing to search for a truth. It will return an error message. Using Or is a little more straightforward, as it's typed out in the code. While And seems to be implied with indentation.

Max Keyword

What if we wanted to go bigger than the above-mentioned function "bigger"? What if we wanted the biggest? We have several options to get there, as usual. But one really handy way: the max code.

def biggest(x,y,z): if x > y: if x > z return x else: return z else: if y > z return y else: return z print biggest(2,9,7) >>> 9 print biggest(3,2,9) >>> 9 print biggest(9,9,3) >>> 9


Efficient Way: Predefined Function def biggest(x,y,z): return bigger(bigger(x,y),z) print biggest(2,9,7) >>> 9 print biggest(3,2,9) >>> 9 print biggest(9,9,3) >>> 9 Best Way: Max def biggest(x,y,z): return max(x,y,z) print biggest(2,9,7) >>> 9 print biggest(3,2,9) >>> 9 print biggest(9,9,3) >>> 9

Unit Two.Five

Debugging

Errors and glitches (Bugs) are a normal part of programming, so the best way to minimize down-time and maximize both efficiency and productivity is to have a systematic way of finding and fixing them. An important side note: when something doesn't behave properly, the errant code doesn't have to be useless. Unexpected results can lead to a new way of doing something else. Theoretically.

Reading the actual error message will help pinpoint the mistaken code, as well as define what kind of error it is. Further, "Googling" the error will lead to possible solutions.

When re-purposing a snippet of code from another source doesn't work correctly, try commenting out the broken code and copying in the source code. This not only makes sure that the source is viable, but allows for easy rewriting with minimized chance for error.

If code behaves oddly try adding in print statements until you come to the line that doesn't work as intended.

Keep multiple versions, and check for errors as you edit. Undo doesn't quite do the trick. A spare file will do just as well as powerful programs designed for version organization.

Mutation

Redefining a variable doesn’t change the value of the original definition, just removes our ability to access it. Strings are not mutable, lists are. If two variables refer to the same list, changing a value in one will change the value of the other.

Replacing values in lists: stooges = ['Moe','Larry','Curly'] # Write one line of code that changes # the value of stooges to be: ['Moe','Larry','Shemp'] # but does not create a new List object. stooges[2]='Shemp' print stooges >>>['Moe','Larry','Shemp']
Aliasing

Having multiple ways to refer to the same value is call Aliasing. If a variable is assigned to another variable that is assigned a value, the first variable has the value of the second variable. Changing one will change the other.

Agent = [‘j’,’a’,’m’,’e’,’s’] Spy = Agent Agent = [0,0,7] print Spy print Agent >>>['j', 'a', 'm', 'e', 's'] >>>[0, 0, 7] # Spy still points to the original value # But Agent points to a new one Agent = [0,0,7] Spy = Agent Agent [2] = Spy[2] + 1 print Spy, Agent >>>[0,0,8] [0,0,8]
